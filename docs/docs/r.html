Object: r

r

r is a JS Object with diferent Resources to work with four data types: HTMLElements, Strings, Arrays and Objects.

test → {HTMLElement|undefined}

test is a property to prove Rocfor is working in the project. It returns a simple string saying: "welcome to Rocforjs!".
Returns:
- String.
Example
let x = new R()
console.log(x.r.test) // "welcome to Rocforjs!"

DOM methods:

$(idname) → {HTMLElement|undefined}

$ is a shortcut for document.getElementById. You can apply the same treatment with r in R.Component.
Parameters:
Name Type Description
idname String Any string of DOM element id attribute.
Returns:
- The HTMLElement Object found in DOM or undefined if doesn't.
Type
HTMLElement | undefined
Example
// Direct assingnment:
let x = new R()
let e = x.r.$('some_id_in_DOM_element') // <div id="some_id_in_DOM_element">Hello world!</div>.
// In a class:
class MyClass extends R {
	constructor (props) {
		super(props)
		let e = this.r.$('some_id_in_DOM_element') // <div id="some_id_in_DOM_element">Hello world!</div>.
	}
}
// In a class that inherits from R.Component:
class MyClass extends R.Component {
	constructor (props) {
		super(props)
		let e = this.r.$('some_id_in_DOM_element') // <div id="some_id_in_DOM_element">Hello world!</div>.
	}
}

elementInsertChild(elem, whereopt) → {Object}

Inserts an HTMLElement into, before or after the HTMLElement specified in DOM.
Parameters:
Name Type Attributes Default Description
elem HTMLElement | Object | String The id of the HTMLElement or the DOM HTMLElement.
where HTMLElement | Object | String <optional>
 body The id of the HTMLElement or the DOM HTMLElement where elem will be inserted.
This param is special, it can receive diferent type of values:
  1. If has a invalid value: body will receive 'elem'.
  2. If it's String: the method will search the HTMLElement in DOM by it's id
  3. If it's HTMLElement object: the method will evaluate it's value and then will insert inside 'elem' HTMLElement.
  4. If it's Object: It can have inside one of three possible elements:
    1. {'container': id string of HTMLElement or the HTMLElement}: 'elem' will be inserted into 'container'
    2. {'before': id string of HTMLElement or the HTMLElement}: 'elem' will be inserted before HTMLElement value of 'before'
    3. {'after': id string of HTMLElement or the HTMLElement}: 'elem' will be inserted after HTMLElement value of 'after'
Returns:
- r Object to chain pattern. If elem or where are invalid datas, an error will be generated.
Type
Object
Example
// Consider next HTML Element in DOM: <div><div id="im"></div></div>
// And let i = <input type="radio" /> the HTMLElement that must be inserted (it is not a string):
// With String params:
this.r.elementInsertChild(i, 'im') // <div><div id="im"><input type="radio" /></div></div>
// With HTMLElement Objects params, consider tags are HTMLElement Objects, not strings:
// let i = <input type="radio" />, myholder = <div><div id="im"></div></div>
this.r.elementInsertChild(i, holder) // <div><div id="im"><input type="radio" /></div></div>
// With 'where' as an object with 'container' property and this HTML Element exists in DOM: <div><div id="im"></div></div>:
// let i = <input type="radio" />
this.r.elementInsertChild(i, {'container': 'im'}) // <div><div id="im"><input type="radio" /></div></div>
// With 'where' as an object with 'before' property
this.r.elementInsertChild(i, {'before': 'im'}) // <div><div id="im"></div><input type="radio" /></div>
// With 'where' as an object with 'after' property
this.r.elementInsertChild(i, {'after': 'im'}) // <div><input type="radio" /><div id="im"></div></div>
