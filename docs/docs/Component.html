Object: Component

Component

Component is a method of R.Component that allows creation of HTMLElement Components. It is created from a string parameter called componentString, this is a group of tagstrings with a tab indentation (as blockStrings); this is important. To create a Component HTMLElement all Tag's basis are applicable, so, in a componentString it is possible to send function callbacks, scope, rvars, props and Markdown contents. In a Component, just one main Element can hold all child Elements, if a siblings Elements is sended, Component will wrap all into a div Element by default. In Component, each Element can have child Elements. As in Tag, Component can be initialized using two ways, the indirect case (assignin Component at any variable: this.myComp = this.Component()), where Component will return an JS Object with a method called createComponent. Or sending the componentString in the assignment in wich case Component will return a new Component Object instance.

Example

// As in Tag, it is possible to declare a new variable assigning Component. (Note it is not important declare a new Component()).
class TestComponent extends R.Component {
	constructor (props) {
		super(props)
		this.mycomp = this.Component() // Direct assignment. Now is possible create a new Component Object in any method of current class.
		console.log(this.mycomp)
		// Console will log:
		{}
		__proto__:
		createComponent: ƒ (componentString, fn, scope, rvars, props)
		__proto__: Object
	}
}

classlist

classlist is an Array of strings, each string is a class used in whole Component final HTMLElement (included Markdown contents). The purpose of this property is to facility all classes applied in each Element of final HTMLElement. Rocforjs do not creates .html files to be used with some tool that selects all don't used classes of some css framework, but, with this property it is possible to create a hidden HTML tag that holds all those classlist allowing the possibility of exclude all don't used classes from css. It is util (per example) when project is created under NPM ambient.
Properties:
Name Type Description
classlist Array | null All classes used in each Component's HTMLElements.

Methods

compChild(index_or_id_childopt) → {HTMLElement|undefined|null}

compChild is a Component's method with wich can get all Component final HTMLElement children or just one of it.
Parameters:
Name Type Attributes Description
index_or_id_child String | Number <optional>
 The id or index of searched Element.
Returns:
- An HTMLElement child or undefined if param is wrong or null if Component does not have any child Element.
Type
HTMLElement | undefined | null
Example
this.mycomp = this.Component(`
Hello
	World!
	#greet Hello Rocforjs!`)
// It works with the child index (first Component's child has index 0):
this.mycomp.compChild(0) // <div>World!</div>
// It works with the child id (Component will search on all children Elements and it's child Elements too):
this.mycomp.compChild('greet') // <div id="greet">Hello Rocforjs!</div>
// Or it will return an Array list with all child Component's HTMLElements:
this.mycomp.compChild() // [<div>World!</div>, <div id="greet">Hello Rocforjs!</div>]

innerHTML(htmltxt, index_or_id_childopt) → {Object}

innerHTML is a special Component's method with wich can set HTML content to main Component's Element or to the selected child Element.
Parameters:
Name Type Attributes Default Description
htmltxt String Any string.
index_or_id_child String <optional>
 component Id or index of child Element searched.
Returns:
Component - To chain pattern.
Type
Object
Example
this.mycomp = this.Component(`
Hello
	World!
	#greet Hello Rocforjs!`)
// It works with the child index (first Component's child has index 0):
this.mycomp.innerHTML('Roccou!', 0) // Now the content of child with index 0 changes to: <div>Roccou!</div>
// It works with the child id (Component will search on all children Elements and it's child Elements too):
// The content of child with id 'greet' will change to: <div id="greet">Hello Roccou!</div>
this.mycomp.innerHTML('Hello Roccou!', 'greet')
// Or it will change all HTML content of main Component Element:
this.mycomp.innerHTML('Hello Roccou!')
// Now the content of main Component changes to: <div>Hello Roccou!</div>
