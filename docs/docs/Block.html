Object: Block

Block

Block is a method of Rocforjs (and R.Component too) that allows creation of HTMLElement Block. It is created from a string parameter called blockString, this is a group of tagstrings with a tab indentation; this is important. To create a Block HTMLElement all Tag basis are applicable, so, in a blockString it is possible to send function callbacks, scope, rvars, props and Markdown contents. In a Block, just one main Element can hold all child Elements, if sibling Elements is sended, Block will wrap all into a div Element by default. In Block, each Element can have child Elements. As in Tag, Block can be initialized using two ways, the indirect case (assignin Block at any variable: this.myBlock = this.Block()), where Block will return an JS Object with a method called createBlock. Or sending the blockString in the assignment in wich case Block will return a new Block Object instance.

Example

// As in Tag, it is possible to declare a new variable assigning Block. (Note it is not important declare a new Block()).
class TestBlock extends R {
	constructor (props) {
		super(props)
		this.myblock = this.Block() // Indirect assignment. Now is possible create a new Block Object in any method of current class.
		console.log(this.myblock)
		// Console will log:
		{}
		__proto__:
		createBlock: ƒ (blockString, fn, scope, rvars, props)
		__proto__: Object
	}
}

// The direct way is sending blockString directly in assignment:
...
this.myblock = this.Block(
`Hello
	World!`) // See how blockString is indented with a tab space and its tag name are ommited too.
consol.log(this.myblock)

classlist

classlist is an Array of strings, each string is a class used in whole Block final HTMLElement (included Markdown contents). The purpose of this property is to facility all classes applyied in each Element of final HTMLElement. Rocforjs do not creates .html files to be used with some tool that selects all don't used classes, but, with this property it is possible to create a hidden HTML tag that holds all those classlist allowing the possibility of exclude all don't used classes from the css framework used in the project. It is util when project is created under NPM ambient.
Properties:
Name Type Description
classlist Array | null All classes used in each Block's HTMLElements.

Methods

blockChild(index_or_id_childopt) → {HTMLElement|undefined|null}

blockChild is a Block's method with wich can get all Block final HTMLElement children or just one of it.
Parameters:
Name Type Attributes Description
index_or_id_child String | Number <optional>
 The id or index of searched Element.
Returns:
- An HTMLElement child or undefined if param is wrong or null if Block does not have any child Element.
Type
HTMLElement | undefined | null
Example
this.myblock = this.Block(`
Hello
	World!
	#greet Hello Rocforjs!`)
// It works with the child index (first Block's child has index 0):
this.myblock.blockChild(0) // <div>World!</div>
// It works with the child id (Block will search on all children Elements and it's child Elements too):
this.myblock.blockChild('greet') // <div id="greet">Hello Rocforjs!</div>
// Or it will return an Array list with all child Block's HTMLElements:
this.myblock.blockChild() // [<div>World!</div>, <div id="greet">Hello Rocforjs!</div>]

delBlock() → {Object}

delBlock is a Block's method with wich can delete Block's HTMLElement previously rendered in DOM.
Returns:
- Block to chain pattern.
Type
Object
Example
// First block HTMLElement must been rendered in DOM, then is possible use this method.
this.myblock = this.Block('Hello World!')
this.myblock.setBlock() // myblock HTMLElement is setted in body's HTMLElement.
this.myblock.delBlock() // myblock HTMLElement is deleted from body's HTMLElement.

innerHTML(htmltxt, index_or_id_childopt) → {Object}

innerHTML is a special Block's method with wich can set HTML content to main Block's Element or to the selected child Element.
Parameters:
Name Type Attributes Default Description
htmltxt String Any string.
index_or_id_child String <optional>
 block Id or index of child Element searched.
Returns:
Block - To chain pattern.
Type
Object
Example
this.myblock = this.Block(`
Hello
	World!
	#greet Hello Rocforjs!`)
// It works with the child index (first Block's child has index 0):
this.myblock.innerHTML('Roccou!', 0) // Now the content of child with index 0 changes to: <div>Roccou!</div>
// It works with the child id (Block will search on all children Elements and it's child Elements too):
// The content of child with id 'greet' will change to: <div id="greet">Hello Roccou!</div>
this.myblock.innerHTML('Hello Roccou!', 'greet')
// Or it will change all HTML content of main Block Element:
this.myblock.innerHTML('Hello Roccou!')
// Now the content of main Block changes to: <div>Hello Roccou!</div>
