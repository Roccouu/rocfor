Object: SPA

SPA()

SPA is a Method of Rocjs that allows creation of a Single Page Application project to works with frontend, backend or both.
This method returns a SPA Object. To start the work with SPA, first use the method setRoutes, then setControler or setControllers.

Methods

changeContainer(controllers_list, controller)

This method allow to change the view container. It is favorable when you want to choose otrher section of your app
as the view container.
Parameters:
Name Type Description
controllers_list Object A Object list with controllers.
controller function | Object The controller as a function callaback or a whole JavaScript Object.
Example
// Assumin' original view container is a DOM HTMLElement with id 'main' and now
// container will be a DOM HTMLElement with id 'aside-holder':
let myspa = this.spa.setRoutes({ ... }, 'main').setControlers(...);
myspa.changeContainer('aside-holder');

playRoute(controllers_list, controller)

This method brings a JavaScript Object with the current route and it's hash parameters if it exists as route/class/method/param1/param2/... If your template is in backend, this method send this object via Ajax.
Parameters:
Name Type Description
controllers_list Object A Object list with controllers.
controller function | Object The controller as a function callaback or a whole JavaScript Object.
Example
// Assumin' some route has next value: 'login' and the navigator route path bar changes to this route: 'login/a/b/c',
// playRoute will obtain all parameters in an object and return current controller and param:
let myspa = this.spa.setRoutes({ ... }, 'main').setControlers(...);
console.log(myspa.playRoute())
// {
// 	'login': {...}, // The controller.
// 	'param': {'route': 'login', 'class': 'a', 'method': 'b', 'params': ['c']} // Route path params.
// }

setController(name, controller)

Optional, SPA controllers setter, waits just for one controller name & object to each route.
This method must been called before all routes are setted.
Parameters:
Name Type Description
name String The name of the controller.
controller function | Object The controller as a function callaback or a whole JavaScript Object.
Example
// Assumin' all routes are setted and one has 'home' controller and other has 'about' controller:
this.spa.setRoutes({ ... })
.setControler('home', {'create': function (){...}, 'read': function () {...}})
.setControler('about', function (){...});

setControllers(controllers_list)

Same setController, but here controller can be a list of objects. This method is optional too,
but always you must set your controllers, so, you has the option to use one of these two methods: setController or setControllers.
Important: Each controller name must coincide with the name controller gived in setRoute.
Parameters:
Name Type Description
controllers_list Object A Object list with controllers.
Example
// Assumin' all routes are setted and one has 'home' controller and other has 'about' controller:
this.spa.setRoutes({ ... })
.setControlers({'home': {'create': function (){...}, 'read': function () {...}}, 'about': function (){...}});

setRoutes(routes, view, default_routeopt)

Waits for a list of routes objects, any html container element that exists in DOM (optional) and the name of a default route.
Parameters:
Name Type Attributes Default Description
routes Object A list of routes.
view String | HTMLElement An HTMLElement or it's id. Here, SPA will allocate the templates.
default_route String <optional>
 '/' An String with the name of default route of the SPA Project.
Example
// Routes is a JSON Object list with all routes for your app.
// Each route would have trhee elements: template, controller, callback, all optional datas.
// Routes must to have at least one route, if you want to set a default route default_route param is for this,
// If you don't set it, SPA assumes '/' as the default route, if it does'nt exists in your list SPA will trhow error.
// Example, assumes code is into a class:
this.myspa = this.spa.setRoutes({
	'/': {'template': './templates/home.html', 'controller': null, 'callback': () => alert('Welcome!')}, // root route.
	'login': {'template': 'Hi! here will appears a Login form!'} // login route.
},
'wrapper') // SPA views container.

// setRoutes has some things to consider:
// 1. It is UNNECESSARY write slash '/' in routes at beginin' or end, example: this/is/my/route.html (just in .html templates)
// 2. By importance order, Template goes first.

// Routes can be writed in many ways, but it are classified in two methods:
// 1. Common and implicit (where SPA asummes the route as the value of templates or controllers) routes elements:
	SPA.setRoutes({
		'/': {'template': 'my/tpl.html' || HTMLElement, 'controller': 'myctrl', 'callback': () => {}}, // This is a common
// route with its template, controller and callback elements.
		'contact': {'callback': function(){ ... }}, // This is an implicit route, SPA understands that route contact has:
// template: './contact.html' and contoller: 'contact', obviously, callback: function(){ ... }
		'info': {'ctrl': 'myctrl'}, // Implicit route just for element template as well: template: './info.html', callback: null
		'about': {}, // Implicit route too, SPA understands: about: {template: './about.html', controller: 'about', cb:null} }
	})
// 2. For null route elements:
	SPA.setRoutes({
		/: {tpl: 'home.html', ctrl: null, cb: null}, // SPA understands: '/': {template: './home.html', 'ctrl': null, 'cb': null}
		contact: {ctrl: null, tpl: null, cb: function(){}}, // For SPA, this means: 'contact': {'tpl': null, 'ctrl': null, 'cb': cb}
		info: {ctrl: null, tpl: info.html, cb: function(){}}, // 'info': {'tpl': './info.html', 'ctrl': null, 'cb': cb}
		about: {ctrl: null, tpl: null, cb: null} // route: will be eliminated, 'cause all it's elements are null.
	})
// If some parameter is wrong, SPA will throws a new error and then, application will be aborted.
// You can send as template four types of values:
// 1. A simple String (it includes HTML tag strings). Example: 'This is a simple string' or '<h1>This is a tag string</h1>'
// 2. A String containing the route path of an html template. Example: './templates/home.html', SPA will request this template via Ajax.
// 3. An HTMLElement, in this case, SPA will append HTMLElement into the view container.
// 4. A String containing the route path of any backend file as .php. Example: './templates/home.php', SPA will request this file via Ajax.
// And if route has parameters it will be sended to backend file as JSON object via POST method on 'params' value.
// If you want to SPA delivers params data to the backend file, its necessary specify that this information
// must go to .php file by a SPA reserved string: '[TOBKND]: ./the/path/to/file.php'. Example:
	SPA.setRoutes({
		/: {template: 'home.html', ctrl: null, cb: null}, // SPA understands: '/': {template: './home.html', 'ctrl': null, 'cb': null}
		contact: {template: '[TOBKND]: ./backend/contacts.php', controller: 'contact', callback: null}
	})
// When navigator's route path bar have the route 'contact' and its parameters for example: 'contact/myclass/myclassmethod/param1'
// Automatically SPA will create a JSON Object with this data:
// {"route":"contact", "class":"myclass","method":"myclassmethod", "params": ["param1"]}
// And when reads the reserved string '[TOBKND]: ...', will send via Ajax and using POST method this data to .php file,
// Where its easy to receive the information via: json_decode($_POST['params'], true); the own php's sentence.
