Object: Tag

Tag

Tag is a method or Rocforjs that allows creation of Single Virtual HTMLElements Objects.
Tag is the core of Rocforjs, and The Library was inspired in Tag.
Tag returns a instance of Tag class when a new Tag is created. Tag requieres four basic parameters: tagString a special Rocforjs's type string with wich is possible build HTMLElements, fn is a param that allows bind any callback at tagString's events, scope to say at Tag that is working in a specific JS class, rvars to work with Rocforjs variables inside tagStrings, and alternatively props to send some properties to Tag when a new class is created.

About fn and scope parameters: What if an HTMLElement must to have an event, or more than one event?. How it is possible that a tagString can hold events to create a new HTMLElement with events associated?
In this land enters the 'fn' Tag parameter.
So, previously, to put events in a tagString, exists some ways that resumes next table:

All invalid or reppeated event writed in tagString will be rejected, so, in final 'tagstring' it's wrong events are discarted as in entirely final HTMLElement.
Now, tagString holds its events and Tag will render this into a new HTMLElement, there is a particular own characteristic of Rocforjs: The final HTMLElement will have its events by the association of the callback and the event throwgh an 'eventListener', so, it will works in a "hide" ambient and the event and it's callback never will be showed in the final HTMLElement (it is easy to recognize, just see in DOM the HTMLElement structure, it could have it's attributes like id, class, title, etc. but no one event as onclick or onmousemove, etc). This is the principle purposed by Rocforjs: "HTML codes and JS codes never have been mixed".
To know what event has the HTMLElemnt Tag has a property called 'events'; if the HTMLElement has not events, events Object will be null. If the HTMLElement created has inline child elements, each child resides in property type Array called 'childtags' and each child holds its own 'events' JS Object. (Note: When use Tag, always, just one main HTMLElement will be created, with or without child HTMLElements).
As the JS method 'addEventListener' assigns a functionality at determined event, JS has a 'removeEventListener' method to delete the indicated event, so, the event gived to the HTMLElement with the tagString can always be deleted ussing 'removeEventListener' method, but, the resident event in 'events' JS Object will exists yet. In fact, this data is only for information it do not has any functionality predeterminated by Tag or Rocforjs because the event is already associated at final HTMLElement, but if once the event is removed from it, but if is necessary, it is possible to reassign this functionality applying again an 'addEventListener' to the HTMLElement and using the 'events' JS Object that holds its calback function, but this must been realized manually.
About rvars: What if in the moment of setting a tagString exists the need to send a variable?. After all, a tagString is just a simple and common string, so, how is possible to send variables into a string?.
With ES6 and it's powerful tools this task is easy: Just use Template Strings. But if this solution is insufficient...
Here enters the concept of rvars. This keyword is a achronym of Rocforjs Variables.
An rvar is a variable with own characteristic inserted into a string and could have a global or a particular reference. References are the way of how the rvar in a string can be identified as a variable to be evaluated as it. Then, a global reference is a rvar inserted into a string and can be evaluated directly by algorithm that searches its value against all superior scopes of the current scope. A particular reference is a kind of reference that uses the parameter 'rvars' of Tag and was thinked to variables for current scope, this parameter can takes one of two methods: first, just a direct reference to only one variable; second: a multiple reference to many variables in current or superiors scopes. Next Table resume it:

rvars Syntax Types For global scopes For local scopes Example Description
$rvar Yes No let tag = this.Tag("p#my-p Testing a $x") (NOT RECOMENDED) $x references a global variable or a variable in superior scope, it's name is 'x' and hold some string value as: "Rocforjs rvars!". Then the tagString resultant wil be: "p#my-p Testing a Rocforjs rvars!". This kind of rvar do not need to be referenced in rvars param.
$$rvar Yes Yes let tag = this.Tag("p#my-p Testing a $$y", {'y': 'Rocforjs rvars!'}) This kind of rvar was thinked to references particulary local variables on current scope, but it works for superior scopes and global variables too (For this task value 'Rocforjs rvars!' in {'y': 'Rocforjs rvars!'} would been changed by the variable reference, example: {'y': x}, in this case key 'y' references at variable 'x' of first kind of rvars). The requirement is that rvars param must to be inserted into a JS Object. This allows the possibility to use many rvars in tagString.
$(rvar) No No let tag = this.Tag("p#my-p Testing a $(1 + 2)") (NOT RECOMENDED) This kind of rvars is just to make mathematical operations. Any other value passed will generate error that console will log.

Rocforjs has a internal algorithm that resides in a JS Object called _Vars. This object allows the evaluation of variables that resides in strings; so, the Rocfor.js Modules that has this functionality are: Mdown, Tag, Block and Component.

Properties

classlist

classlist property is an Array that holds a list of all classes of HTMLElement created by "Tag" included it's child Elements and Markdown content. This property was thonked on the necessity of know which classes alredy have the current element and all its children; this serves when uses a tool that helps to select just all classes used in current project and deletes all that don't be used.
Properties:
Name Type Description
classlist Array | null All Tag HTMLElement classes, included Tag's child HTMLElements.

events

events property is a JS Object and holds all events of the final HTMLElement. it is null if do not have events.
Properties:
Name Type Description
events Object | null All Tag events list.

idDOM

idDOM property is an Array that holds a coincident list of the HTMLELements id's of DOM and Tag's created HTMLElement and it's child Elements. If this property comes to have any id inside the Array, Tag will log a warn on console warning that one or more id's already exists in DOM. And if the error persists, Tag will abort process returning a null value.
Properties:
Name Type Description
idDOM Object | null All DOM's id repeated in Tag.

idlist

idlist is an Array that holds a list of HTMLElement id and all it's child Elements id's, null otherwise.
Properties:
Name Type Description
idlist Array | null All id's of each HTMLElements in Tag.

tag

tag property is The Final HTMLElement.
Properties:
Name Type Description
tag HTMLElement | null The final Tag HTMLElement.

tagchild

tagchild property is an Array of JS Objects that holds a list of all child Elements of final HTMLElement. Each JS Object holds next properties: element, it's value is an HTMLElement; events, an JS Object with a list of current HTMLElement, tagstring, the current HTMLElement own tagstring.

tagstrings

tagstrings is a JS Object that holds two elements: tagstring and mytagString. Each element holds in its value a string. The tagString element holds the value of tagString param sended in Tag (or createTag) and tagstring element holds a string created for Tag algorithm and show the correct way of how the tagString must be writed. Both elements are datas for programmer's information.

About 'tagString': Rocforjs gives this name at this parameter because this string is a special type of string, as you know, to write HTML code, it's necessary to write HTML tags (example: <p id="my-p">Hello</p>), and then, with this tag, navigators will understand that must to create HTMLElements and render it into the DOM.
Rocforjs allows an easy, rapid and simple way to creation of tags, supressing the use of tags symbols (<, / and >) and in base on the HTML tags concept of: "each tag must have a tag name, attributes (optional) and content (optional too)", each HTML tag now can be a simple string that could be converted in a new HTML tag.
So, Rocforjs introduces the "tagString" concept, then, a tagString is a common but special type of string, because this kind of string represents an HTML tag. This is the reason of it's name: "tagString".
To write any tagString, you must to write first the optional HTML tag name followed by it's optional attributes and finally it's optionally too: content.
Example:
  1. Tag name: p
  2. Tag attributes: (id="my-p")
  3. Tag content: Hello
  4. ...
  5. All in one: p(id="my-p") Hello

But, as tagStrings are so simple to write, if your tagString is as simple than the tagString 'Hello World!' of the first and second examples abobe, Tag understands that tagString 'Hello World!' haven't attributes, just content, (tag name is optional). Tag defines that for all tagStrings, it's default tag name for block elements is 'div' and for inline elements is 'span', then, the correct tagString in the example would be: 'div Hello World!'.
This correct tagString is builded by Tag internally, in it's core algorithm and the result is called "tagstring", then, always, Tag will return a JS Object "tagstrings" with two elements: "mytagString" and "tagstring", obviously, "mytagString" is the "tagString" data passed by Programmer when create a new Tag, and "tagstring" is the data that Tag build internally.

Now you know what tagString and tagstring are, and how can write tagStrings, let define how to write it:

  1. Id's and classes optionaly, can be writed inline before tagname preceded by symbols # (hash) to id and . (dot) to classes:
    1. p#my-p
    2. p#my-p.My-p-class
  2. Other attributes must been writed into parenthesess, each attribute separed by commas and one space and it's value beween "":
    1. p#my-p.My-p-class(title="A title", data-mine="my-data", click="function () {alert(\'Hi!\')};")
  3. As, tagname, attributes, and content are optionals in a tagString, followin' tagStrings are valid:
    1. "Hello World!"
    2. "p Hello World!"
    3. "p#my-p Hello World!"
    4. "p.My-p-class Hello World!"
    5. "p(id="my-p", class="My-p-class") Hello World!"
    6. "(id="my-div", class="My-div-class") Hello World!"
    7. "#my-div Hello World!"
    8. ".My-div-class Hello World!"
    9. "#my-div.My-div-class Hello World!"
    10. "#my-div.My-div.center.box Hello World!"
    11. "(id="my-div", class="My-div,center box.square, big") Hello World!"
  4. To write tagStrings that holds inline tags, write common tagString between symbols #[mytagString]:
    1. "p Hello World! #[em.square And Hello Rocforjs! #[And Hello Roccou! ]]#[strong And Welcome Tag!]"
    2. Under this example, is possible to write many inline tags as the imagination permits.
  5. Tag supports Markdown contents!, it because Rocforjs incorporates a simple and "artesanal" algorithm wrote to perform fast HTML contents (See Mdown Object for more information):
    1. "p MD: #Hello World! #[em.square And Hello Rocforjs! #[And MD: *Hello Roccou!* ]]#[strong And Welcome Tag!]"
When MD: keyword is used in Tag (Block or Component) tagstring in JS Object tagstrings will adopt a remarked content ([Markdown content/]), not the entirely Markdown content because it's length could be so biggest to be managed as a single property value.
Properties:
Name Type Description
tagstrings Object | null The Tag generated tagstring and Programmer writed mytagString.
Example
console.log(mytag.tag.tagstrings)
// Console will log:
tagstrings:
	mytagString: "Hello World!]"
	tagstring: "div Hello World!"

Methods

delTag() → {Object}

With this method final HTMLElement previously rendered in DOM can be deleted.
Returns:
- Tag object to chain pattern.
Type
Object
Example
this.tag = this.tag.createTag('Hello World!')
this.tag.setTag()
this.tag.delTag() // HTMLElement will be removed from DOM.

getTag() → {HTMLElement}

With this method is possible to get a final HTMLElement.
Returns:
- The final HTMLElment creted with Tag.
Type
HTMLElement
Example
this.tag = this.tag.createTag('Hello World!')
this.tag.getTag() // <div>Hello World!</div>.

setTag(whereopt) → {Object}

With this method, the final HTMLElement can be rendered on DOM.
If where param is ommited, by default, setTag will render final HTMLElement in body or in container determined in props as a id string of an HTMLElement that exists in DOM or JS Object with one of three possible keys: {['after'|'container'|'before']:'id_or_HTMLElemnt'}.
Parameters:
Name Type Attributes Default Description
where String | HTMLElement <optional>
 'body'|'props['after|container|before']|props' The DOM HTMLElement that will hold new HTMLElement.
Returns:
- Tag object to chain pattern.
Type
Object
Example
this.tag = this.tag.createTag('Hello World!')
this.tag.setTag() // HTMLElement will be rendered in DOM's body.

tagChild(index_or_id_childopt) → {HTMLElement|Array|null}

With this method is possible to obtain one or all children Elements of a final HTMLElement.
Parameters:
Name Type Attributes Description
index_or_id_child String | Number <optional>
 The id or index of child element wanted.
Returns:
- An HTMLElement or an Array of HTMLElements or null if main Element does not have children.
Type
HTMLElement | Array | null
Example
this.tag = this.tag.createTag('Hello World! #[span#im I\'m a child Element. ]#[em Me too.]')
this.tag.tagChild('im') // <span id="im">I'm a child Element. </span> // It works with id's
this.tag.tagChild(2) // <em>Me too.</em> // It works with Array index.
// Or if do not exist param, a list with all children is returned.
this.tag.tagChild() // [<span id="im">I'm a child Element. </span>, <em>Me too.</em>]

tagstringChild(index_childopt) → {String|Array|null}

With this method is possible to obtain one or all tagstring of child Elements.
Parameters:
Name Type Attributes Description
index_child Number <optional>
 The index of child element wanted.
Returns:
- A string or an Array of strings or null if main Element does not have children.
Type
String | Array | null
Example
this.tag = this.tag.createTag('Hello World! #[span#im I\'m a child Element. ]#[em Me too.]')
this.tag.tagstringChild(2) // "em Me too." // It works just with Array index.
// If do not exist param, a list with all tagstring children is returned.
this.tag.tagstringChild() // ["Hello World!", "span#im I\'m a child Element. ", "em Me too."]
