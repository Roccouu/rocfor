Mdown

Mdown is a short Markdown contents translator to HTML. Do not support nested elements as p or ul in blockquotes.
Supports classes: class(w,x y.z), and id's: id(myid). It was spetialy thinked for developers.

Example

let mymd = Mdown('# I\'m an H1') // <h1>I'm an H1</h1>
let mymd = Mdown('id(myh1)# I\'m an H1') // <h1 id="myh1">I'm an H1</h1>
let mymd = Mdown('class(Foo.Bar Baz-baz)# I\'m an H1') // <h1 class="Foo  Bar  Baz-baz">I'm an H1</h1>
let mymd = Mdown('id(myh1)class(Foo.Bar Baz-baz)# I\'m an H1') // <h1 id="myh1" class="Foo  Bar  Baz-baz">I'm an H1</h1>
// Use ? (with one space ending) to create a div tag and ? (with two space ending) to close div:
let mymd = Mdown(`
'id(mydiv)class(Foo)? 
# I'm an H1 inside a div with id mydiv.
?  `) // <div id="mydiv"><h1>I'm an H1 inside a div with id mydiv.</h1></div>
// If you want to introduce backticks (``) in your content, please use (´´) tokens.
let mymd = Mdown('A paragraph ´p´ in my content.') // <p>A paragraph `p` in my content.</p>
// Lists ul or ol must to have one space ending in open tag and two spaces in close tag as ? on div tag.

// It supports 'rvars':
let cont = 'in my content' // a global variable.
let mymd = Mdown('A paragraph ´p´ $cont.') // <p>A paragraph `p` in my content.</p>
let local = 'MY CONTENT' // a local variable.
let mymd2 = Mdown('A paragraph ´p´ $$loc.', {'loc':local}) // <p>A paragraph `p` MY CONTENT.</p>
// To obtain hr tag: ------ or ______ or ======
// For ul lists tags: + or * or - and then any content with one space ending the first line and two spaces the last line.
// For ol lists tags: 1. (any number foolwed by dot and a tab) and then any content with
// one space ending the first line and two spaces the last line.
// For paragraph tags: Just any content for one line, two spaces ending inserts an br tag and three spaces ends p tag.
// For pre tags: ``` or +++ or \`\`\` followed by language name, as: javascript. This language name will be
// added on HTML pre tag as a class. ``` or +++ or \`\`\` to end pre tag.
// For tables: Use | to separe datas as in normal Markdown. First line will be the thead tag and next lines will be
// inserted on tbody tag and in it's tr and td tags.
// For contents with links and images references, use a document divided in two parts: The main content and The references section:
// Example:
// 'This is an example content with a referenced link [Rocforjs][1] and a referenced image: ![alt text][id]
// [//]: # References
// [1]: http://rocforjs.com/	"Rocforjs"
// [id]: /path/to/img.jpg	"Title"
